\begin{abstract}
Image  Aesthetic  Assessment  (IAA)  has  attracted wide interest in recent years, but it remains challenging due to its highly subjective and ambiguous nature. With the prevalence of  social  media,  user   comments  can  provide   rich  aesthetics- aware   semantic   information   and   human   observers   typically evaluate  images  from  the  perspective  of  aesthetic  attributes. Motivated by these findings and the effectiveness of learnable queries in extracting task-specific   features   from   frozen   models,   we   propose   Learnable Query-Guided Visual-Comment Interaction Network (LQ-VCINet) for IAA, which include Learnable Queries Guided Intra-Cross modal Interaction module (LQ-ICI) to perform hierarchical feature fusion, a global token guided multiple layer feature aggregation module (GFA), and a attribute guided representation refinement module (ARR) and a prediction head to perform aesthetic assessment.  Experiments  on  AVA-Comments dataset demonstrate that LQ-VCINet achieves state-of-the-art performance on  multimodal  IAA,  with  an  SRCC  of  0.915  and  a  PLCC  of 0.920.
\end{abstract}
\begin{IEEEkeywords}
Image aesthetic assessment, learnable queries, multimodal fusion, hierarchical interaction, aesthetic attributes.
\end{IEEEkeywords}

\section{Introduction}

\IEEEPARstart{I}{mage} aesthetic assessment (IAA) aims to automatically predict human aesthetic perception of images. Owing to its wide applications in fields such as image retrieval, photo recommendation, automatic enhancement, and social media content management, this task has attracted growing interest in recent years.

Due to its highly abstract nature, IAA remains a challenging research topic. Early IAA approaches focused on designing hand-crafted features based on photographic rules. More recently, with the prevalence of deep learning, distribution prediction methods(such as NIMA)utilize Earth Mover's Distance (EMD) to predict aesthetic score distributions. However, relying solely on aesthetic score labels provides limited supervision information, which may result in a neglect of aesthetic details.

\centering
\includegraphics[width=\columnwidth]{method_comparison.png}
\caption{Comparison of representative image aesthetic assessment paradigms. Unlike visual-only and coarse auxiliary-fusion methods, LQ-VCINet exploits multi-level visual-comment interaction and attribute-guided refinement for aesthetic score distribution prediction.}
\label{fig:method_comparison}
\end{figure}

Based on existing works, user comments can not only provide complementary semantic features, but also describe attribute characteristics. In recent years, a few methods employ aesthetic attributes to connect visual and comment representations. These studies indicate that it is of great value to jointly model visual appearances, comment feedback, and aesthetic attributes.

Recently, large-scale pre-trained models have demonstrated strong representationcapabilities. Learnable queries have also been shown to be effective in extracting useful task-specific features from frozen pre-trained backbones. Therefore, we propose Learnable Query-Guided Visual-Comment Interaction Network (LQ-VCINet) for IAA, which applies learnable query-guided intra-modal and cross-modal feature interaction independently at each of multiple encoder layers to capture multi-level aesthetic representations, aggregates the layer-wise outputs via a global token, and further refines the representation with aesthetic attribute priors.

Specifically, LQ-VCINet uses pre-trained CLIP image encoder, BERT, and an attribute encoder with freezed weights to extract visual, textual and attribute features, and only trains lightweight fusion and prediction modules. The framework first employs learnable queries to extract modality-specific aesthetic features from images and comments, and then models the complementary relationships between images and comments via cross-modal attention. Subsequently, a global token is utilized to aggregate multi-layer features, and the aesthetic attribute priors extracted by the attribute encoder are introduced to refine the final representation, thereby predicting the aesthetic score distribution.

The primary contributions of this paper are summarized as follows:
\begin{itemize}
\item We propose a multi-modal framework LQ-VCINet for aesthetic score distribution prediction. By freezing the pre-trained visual, comment, and attribute encoders, this framework integrates visual, comment, and aesthetic attribute information via lightweight modules.
\item We propose the LQ-ICI module, which applies learnable query-guided intra-cross interaction hierarchically across $L$ encoder layers. At each layer, queries first capture intra-modal aesthetic features and then exchange complementary information cross-modally, enabling more comprehensive aesthetic information interactions.
\item We design  global token guided multiple layer feature aggregation module (GFA) and an attribute guided representation refinement module (ARR) to integrate multi-level aesthetic features and introduce structured aesthetic attribute priors, respectively, thereby enhancing the aesthetic prediction performance.
\end{itemize}

\section{Method}

\subsection{Problem Formulation}

Given the $i$-th image and its corresponding user comment $(I_i,C_i)$, this paper aims to learn a mapping function $f(\cdot)$, which is formalized as:
\begin{equation}
\hat{\mathbf{y}}_i=f(I_i,C_i),
\end{equation}
where $\hat{\mathbf{y}}_i\in\mathbb{R}^{K}$ denotes the predicted aesthetic score distribution, $f(\cdot)$ represents the proposed assessment model, and $K$ is the number of score levels (e.g., $K=10$ for the AVA dataset). The predicted distribution satisfies the normalization constraint $\sum_{k=1}^{K}\hat{y}_i^k=1$. Based on the predicted distribution, the mean aesthetic score of the image can be further computed as:
\begin{equation}
\label{eq:mean_score}
\mu_i=\sum_{k=1}^{K}k\cdot\hat{y}_i^k.
\end{equation}

Although images and their associated comments can provide rich aesthetic information, combining these two different modalities to predict aesthetic score distributions remains challenging. First, due to the subjective nature of image aesthetics, visual features and comment descriptions lack stable semantic correspondence, making cross-modal fusion particularly challenging. Second, human aesthetic perception spans multiple levels from low-level texture and color to high-level composition and semantics, while single-level feature representations fail to capture this full range. Third, without explicit aesthetic attribute priors to guide multimodal fusion, the resulting representations tend to be less discriminative for aesthetic assessment.

To address these challenges, we propose LQ-VCINet and the flowchart is shown in Fig.~\ref{fig:lqicif_overview}, which consisting of three key modules: (1) learnable query-guided intra-cross interaction(LQ-ICI), (2) global token guided multiple layer feature aggregation module (GFA), and (3) attribute guided representation refinement module (ARR).

\subsection{Learnable query-guided intra-cross interaction}

Instead of directly applying cross-attention across all tokens, we employ learnable queries to first extract intra-modal aesthetic features before performing cross-modal interactions. The scaled dot-product attention is defined as:
\begin{equation}
\mathrm{Attn}(\mathbf{Q},\mathbf{K},\mathbf{V})=
\mathrm{Softmax}\left(\frac{\mathbf{Q}\mathbf{K}^{\top}}{\sqrt{d}}\right)\mathbf{V},
\end{equation}
where $d$ is the feature dimension. For brevity, $\mathrm{Attn}(\mathbf{Q},\mathbf{X})=\mathrm{Attn}(\mathbf{Q},\mathbf{X},\mathbf{X})$ when keys and values share the same source.

Self-attention is defined as a special case where queries, keys, and values all come from the same input:
\begin{equation}
\mathrm{SA}(\mathbf{X})=\mathrm{Attn}(\mathbf{X},\mathbf{X},\mathbf{X}).
\end{equation}
When applying $D$ stacked self-attention layers, we denote it as $\mathrm{SA}^{(D)}(\cdot)$.

For the feature pair $(\mathbf{V}_l,\mathbf{T}_l)$ at layer $l$, where $\mathbf{V}_l$ and $\mathbf{T}_l$ denote the visual and textual features extracted from the $l$-th layer of CLIP image encoder and BERT respectively, learnable queries $\mathbf{Q}_l^V,\mathbf{Q}_l^T\in\mathbb{R}^{N_q\times d}$ (where $N_q$ is the number of queries) first aggregate intra-modal aesthetic features via cross-attention:
\begin{equation}
\begin{aligned}
{\mathbf{R}}_l^V&=\mathrm{Attn}(\mathbf{Q}_l^V,\mathbf{V}_l),&
{\mathbf{R}}_l^T&=\mathrm{Attn}(\mathbf{Q}_l^T,\mathbf{T}_l).
\end{aligned}
\end{equation}
Cross-modal attention then models the complementary relationship between visual and comment features:
\begin{equation}
\begin{aligned}
\hat{\mathbf{R}}_l^V&=\mathrm{Attn}(\mathbf{R}_l^V,\mathbf{T}_l),&
\hat{\mathbf{R}}_l^T&=\mathrm{Attn}(\mathbf{R}_l^T,\mathbf{V}_l).
\end{aligned}
\end{equation}
The intra- and cross-modal features are aggregated via mean fusion to obtain the layer-$l$ output representations:
\begin{equation}
\mathbf{F}_l^V=\frac{{\mathbf{R}}_l^V+\hat{\mathbf{R}}_l^V}{2},\quad
\mathbf{F}_l^T=\frac{{\mathbf{R}}_l^T+\hat{\mathbf{R}}_l^T}{2}.
\end{equation}
Applying this process across all $L$ layers yields $\{\mathbf{F}_l^V\}_{l=1}^{L}$ and $\{\mathbf{F}_l^T\}_{l=1}^{L}$, progressively capturing hierarchical visual-comment interaction fusion information.

\subsection{ Global token guided multiple layer feature aggregation}

To capture multi-level aesthetic characteristics, we average each layer's output across the query dimension:
\begin{equation}
\mathbf{Z}_l^V=\mathrm{Mean}(\mathbf{F}_l^V),\quad
\mathbf{Z}_l^T=\mathrm{Mean}(\mathbf{F}_l^T),\quad l=1,\ldots,L,
\end{equation}
where $\mathbf{Z}_l^V,\mathbf{Z}_l^T\in\mathbb{R}^{d}$. The layer-wise features are then concatenated with learnable global tokens $\mathbf{g}^V,\mathbf{g}^T\in\mathbb{R}^{d}$, and processed by a $D$-layer stacked self-attention module after adding positional encodings:
\begin{equation}
\begin{aligned}
\mathbf{S}^{V}&=\mathrm{SA}^{(D)}([\mathbf{g}^{V};\mathbf{Z}_{1}^{V};\ldots;\mathbf{Z}_{L}^{V}]),\\
\mathbf{S}^{T}&=\mathrm{SA}^{(D)}([\mathbf{g}^{T};\mathbf{Z}_{1}^{T};\ldots;\mathbf{Z}_{L}^{T}]),
\end{aligned}
\end{equation}
where $D=2$ and $\mathbf{S}^{V},\mathbf{S}^{T}\in\mathbb{R}^{(L+1)\times d}$. The first position of each output sequence (i.e., the global token position) is extracted as the holistic visual and textual representation:
\begin{equation}
\mathbf{h}^{V}=\mathbf{S}^{V}[0],\quad \mathbf{h}^{T}=\mathbf{S}^{T}[0].
\end{equation}
Finally, the two modality representations are concatenated to form the multimodal fused feature:
\begin{equation}
\mathbf{F}_{\mathrm{cat}}=[\mathbf{h}^{V};\mathbf{h}^{T}]\in\mathbb{R}^{2d}.
\end{equation}

\subsection{Attribute-Guided Representation Refinement}

For fused $\mathbf{F}_{\mathrm{cat}}$, aesthetic attributes are further introduced to obtain attribute aware features. The attribute encoder extracts a global attribute feature $\mathbf{g}\in\mathbb{R}^{M}$ and $A$ attribute scores $s^1,\ldots,s^A$ (we use 11 aesthetic attributes, thus $A=11$). Each attribute score is concatenated with $\mathbf{g}$ and projected into an attribute token via an attribute projection layer (linear + layer normalization + ReLU), forming the attribute token sequence:
\begin{equation}
\mathbf{A}_{\mathrm{tok}}=
[\mathrm{Proj}([\mathbf{g};s^1]);\ldots;\mathrm{Proj}([\mathbf{g};s^A])]
\in\mathbb{R}^{A\times d}.
\end{equation}
Using $\mathbf{F}_{\mathrm{cat}}$ as the query and $\mathbf{A}_{\mathrm{tok}}$ as keys and values, the fused feature is then calibrated via cross-attention (as defined in Eq.~(3)):
\begin{equation}
\hat{\mathbf{Y}}=\mathrm{Attn}(\mathbf{F}_{\mathrm{cat}},\mathbf{A}_{\mathrm{tok}})\in\mathbb{R}^{2d}.
\end{equation}
The refined feature $\hat{\mathbf{Y}}$ is subsequently fed into the prediction head for aesthetic score distribution prediction.

\subsection{Aesthetic Score Distribution Prediction}

The refined representation $\hat{\mathbf{Y}}_i$ is fed into an MLP head followed by softmax to estimate the aesthetic score distribution:
\begin{equation}
\hat{\mathbf{y}}_i=\mathrm{Softmax}(\mathrm{MLP}(\hat{\mathbf{Y}}_i)),
\end{equation}
The softmax output utilizes the distribution of human scores instead of collapsing them into a single scalar during training. Following distribution-based IAA assessment, the normalized ground-truth $\mathbf{y}_i$ supervises $\hat{\mathbf{y}}_i$ with the Earth Mover's Distance (EMD):
\begin{equation}
\mathcal{L}_{\mathrm{emd}} =
\left(
\frac{1}{K}\sum_{k=1}^{K}
\left|
\mathrm{CDF}_{\hat{\mathbf{y}}_i}(k)-
\mathrm{CDF}_{\mathbf{y}_i}(k)
\right|^r
\right)^{1/r},
\end{equation}
where $\mathrm{CDF}_{\hat{\mathbf{y}}_i}(k)=\sum_{j=1}^{k}\hat{y}_i^j$, $\mathrm{CDF}_{\mathbf{y}_i}(k)=\sum_{j=1}^{k}y_i^j$, and $r=2$. The final mean score is computed by \eqref{eq:mean_score}.

\section{Experiments}
\label{sec:experiments}

\subsection{Experimental Protocols}

We evaluate on AVA-Comments, which provides 3,602,206 comments for 255,530 images.

We report SRCC, PLCC, ACC, and EMD to evaluate ranking consistency, linear correlation, binary classification, and distribution prediction, respectively. The AVA score distribution is directly normalized as ground truth. All experiments are implemented in PyTorch on a single NVIDIA GeForce RTX 5060 Ti GPU.

\subsection{Implementation Details}

All comments matched to an image are concatenated as text input, while missing comments are replaced by a ``no comment'' token. Images are resized to $224\times224$ with random horizontal flipping during training. We freeze CLIP ViT-L/14, BERT-base, and the attribute encoder, and train only the Adapter modules, projection layers, and attention modules for 10 epochs using AdamW with learning rate $1\times10^{-4}$ and batch size 16.

The first three variants ($A_0$--$A_2$) demonstrate the value of multimodal modeling. The single-modality baselines ($A_0$, SRCC 0.757; $A_1$, SRCC 0.819) show that neither visual nor comment features alone are sufficient for accurate aesthetic assessment. Simple concatenation ($A_2$, SRCC 0.884, EMD 0.046) confirms the complementarity of the two modalities, while direct single-layer cross-attention ($A_3$, SRCC 0.912) shows that explicit cross-modal interaction brings clear gains over naive feature merging.

The comparison between $A_3$ and $A_4$ validates the proposed ICIF design. By first capturing intra-modal aesthetic features via learnable queries before cross-modal exchange, ICIF ($A_4$) improves SRCC/PLCC to 0.914/0.920 and ACC to 87.10, confirming that preserving modality-specific structure prior to cross-modal fusion yields richer and more discriminative aesthetic representations.

The full model ($A_5$) incorporates hierarchical multi-layer aggregation and attribute-guided refinement on top of ICIF, achieving the best results across all four metrics (SRCC 0.915, PLCC 0.920, ACC 87.40, EMD 0.041). These gains confirm that both global token aggregation and aesthetic attribute priors contribute meaningfully to the final prediction.

Symbol. If you are supplying EPS, PS, or PDF files all fonts must be 
embedded. Some fonts may only be native to your operating system; without 
the fonts embedded, parts of the graphic may be distorted or missing.

A safe option when finalizing your figures is to strip out the fonts before 
you save the files, creating ``outline'' type. This converts fonts to 
artwork what will appear uniformly on any screen.

\subsection{Using Labels Within Figures}

\subsubsection{Figure Axis labels }
Figure axis labels are often a source of confusion. Use words rather than 
symbols. As an example, write the quantity ``Magnetization,'' or 
``Magnetization M,'' not just ``M.'' Put units in parentheses. Do not label 
axes only with units. As in Fig. 1, for example, write ``Magnetization 
(A/m)'' or ``Magnetization (A$\cdot$m$^{-1}$),'' not just ``A/m.'' Do not label axes with a ratio of quantities and 
units. For example, write ``Temperature (K),'' not ``Temperature/K.'' 

Multipliers can be especially confusing. Write ``Magnetization (kA/m)'' or 
``Magnetization (10$^{3}$ A/m).'' Do not write ``Magnetization 
(A/m)$\,\times\,$1000'' because the reader would not know whether the top 
axis label in Fig. 1 meant 16000 A/m or 0.016 A/m. Figure labels should be 
legible, approximately 8 to 10 point type.

\subsubsection{Subfigure Labels in Multipart Figures and Tables}
Multipart figures should be combined and labeled before final submission. 
Labels should appear centered below each subfigure in 8 point Times New 
Roman font in the format of (a) (b) (c). 

\subsection{File Naming}
Figures (line artwork or photographs) should be named starting with the 
first 5 letters of the author's last name. The next characters in the 
filename should be the number that represents the sequential 
location of this image in your article. For example, in author 
``Anderson's'' paper, the first three figures would be named ander1.tif, 
ander2.tif, and ander3.ps.

Tables should contain only the body of the table (not the caption) and 
should be named similarly to figures, except that `.t' is inserted 
in-between the author's name and the table number. For example, author 
Anderson's first three tables would be named ander.t1.tif, ander.t2.ps, 
ander.t3.eps.

\subsection{Referencing a Figure or Table Within Your Paper}
When referencing your figures and tables within your paper, use the 
abbreviation ``Fig.'' even at the beginning of a sentence. Do not abbreviate 
``Table.'' Tables should be numbered with Roman Numerals.

Because IEEE will do the final formatting of your paper,
you do not need to position figures and tables at the top and bottom of each

\section{Conclusion}

In this letter, we proposed LQ-VCINet, a parameter-efficient multimodal framework for image aesthetic score distribution prediction. By combining learnable query-guided image-comment interaction, hierarchical aggregation, and attribute encoder-based refinement, the proposed method exploits complementary visual, comment, and aesthetic attribute information while keeping pretrained encoders frozen. Experiments on AVA-Comments demonstrate the effectiveness of combining user comments and aesthetic attribute priors for accurate image aesthetic assessment.

\section*{References and Footnotes}

\subsection{References}

References need not be cited in text. When they are, they appear on the line, in square brackets, inside the punctuation.  Multiple references are each numbered with separate brackets. When citing a section in a book, please give the relevant page numbers. In text, refer simply to the reference number. Do not use ``Ref.'' or ``reference'' except at the beginning of a sentence: ``Reference [3] shows . . . .'' Please do not use automatic endnotes in {\em Word}, rather, type the reference list at the end of the paper using the ``References'' style.

Reference numbers are set flush left and form a column of their own, hanging out beyond the body of the reference. The reference numbers are on the line, enclosed in square brackets. In all references, the given name of the author or editor is abbreviated to the initial only and precedes the last name. Use them all; use {\em et al.} only if names are not given. Use commas around Jr., Sr., and III in names. Abbreviate conference titles.  When citing IEEE transactions, provide the issue number, page range, volume number, year, and/or month if available. When referencing a patent, provide the day and the month of issue, or application. References may not include all information; please obtain and include relevant information. Do not combine references. There must be only one reference with each number. If there is a URL included with the print reference, it can be included at the end of the reference.

Other than books, capitalize only the first word in a paper title, except for proper nouns and element symbols. For papers published in translation journals, please give the English citation first, followed by the original foreign-language citation. See the end of this document for formats and examples of common references. For a complete discussion of references and their formats, see the IEEE style manual at www.ieee.org/authortools.

Number footnotes separately in superscripts (Insert $\mid$ Footnote).\footnote{It is recommended that footnotes be avoided (except for the unnumbered footnote with the receipt date on the first page). Instead, try to integrate the footnote information into the text.}  Place the actual footnote at the bottom of the column in which it is cited; do not put footnotes in the reference list (endnotes). Use letters for table footnotes (see Table I).

b) J. K. Author, ``Title of paper,'' unpublished.

c) J. K. Author, ``Title of paper,'' to be published.

\subsection*{Examples:}

\begin{thebibliography}{34}
\setcounter{enumiv}{26}

\bibitem{}A. Harrison, private communication, May 1995.

\bibitem{}B. Smith, ``An approach to graphs of linear forms,'' unpublished.

\bibitem{}A. Brahms, ``Representation error for real numbers in binary computer arithmetic,'' IEEE Computer Group Repository, Paper R-67-85.
\end{thebibliography}

\subsection*{Basic formats for standards:}

a) {\em Title of Standard}, Standard number, date.

b) {\em Title of Standard}, Standard number, Corporate author, location, date.

\subsection*{Examples:}

\begin{thebibliography}{34}
\setcounter{enumiv}{29}


\bibitem{}IEEE Criteria for Class IE Electric Systems, IEEE Standard 308, 1969.

\bibitem{} Letter Symbols for Quantities, ANSI Standard Y10.5-1968.
\end{thebibliography}
